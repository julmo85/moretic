Projet citoyen en construction

Autonomie sans frontières architecturales

Un guide pratique terrain pour mieux documenter l’accessibilité réelle à Fribourg : trajets, WC accessibles, parkings PMR, lieux culturels, obstacles, repos et informations utiles avant une sortie.

Français English Deutsch
Voir les fiches pilotes prévues Voir les sources identifiées

1. Idée générale

Le projet “Autonomie sans frontières architecturales” vise à mieux documenter l’accessibilité réelle à Fribourg.

Il s’adresse d’abord aux personnes en situation de handicap ou à mobilité réduite, mais il peut aussi être utile aux personnes âgées, aux proches aidants, aux accompagnants, aux parents avec poussette, aux personnes temporairement blessées ou à toute personne qui a besoin d’informations fiables pour se déplacer plus sereinement.

L’objectif n’est pas de remplacer les informations officielles existantes, mais de les compléter avec une approche pratique, centrée sur les besoins concrets des personnes concernées.

Une ville accessible ne se résume pas à une rampe ou à un pictogramme. Il faut aussi pouvoir arriver, entrer, circuler, utiliser les toilettes, se reposer, demander de l’aide, repartir et anticiper les obstacles.

2. Constat

Aujourd’hui, certaines informations existent déjà, mais elles sont souvent dispersées, incomplètes ou trop générales pour permettre à une personne de préparer une sortie sereinement.

Pour préparer une sortie, une personne doit pouvoir savoir concrètement :

  • comment arriver ;
  • où se parquer ;
  • quel arrêt de bus utiliser ;
  • s’il y a des pentes ou des pavés ;
  • comment entrer ;
  • s’il y a un ascenseur ;
  • où trouver des toilettes accessibles ;
  • où se reposer ;
  • qui contacter en cas de besoin ;
  • s’il existe un plan B en cas d’obstacle.

3. Ce que le projet apporte

Le projet propose de :

  • centraliser les informations utiles ;
  • identifier les lacunes pratiques ;
  • documenter quelques lieux et trajets concrets ;
  • distinguer les sources officielles, les observations de terrain et les éléments à vérifier ;
  • proposer des améliorations réalistes ;
  • encourager la contribution des personnes concernées ;
  • faciliter la collaboration entre associations, institutions, acteurs touristiques et autorités.

4. Fiches pilotes prévues

Pour commencer de manière concrète et réaliste, le projet pourrait se concentrer sur 3 ou 4 fiches pilotes. Ces fiches permettraient de démontrer rapidement l’utilité de l’approche, sans vouloir documenter toute la ville dès le départ.

Fiche pilote 1

Trajet gare de Fribourg → centre-ville

Objectif : montrer si le trajet est réellement praticable, où sont les pentes, les pavés, les passages difficiles, les arrêts utiles et les lieux de pause.

Fiche pilote 2

WC public accessible

Objectif : vérifier l’emplacement, l’accès, les horaires, l’éventuelle clé Eurokey, le chemin jusqu’aux toilettes et leur utilité réelle pendant une sortie.

Fiche pilote 3

Parking PMR proche d’un lieu important

Objectif : documenter la place, la distance jusqu’au lieu, les pentes, les pavés, l’accès au trottoir et les alternatives possibles.

Fiche pilote 4

Musée ou lieu culturel fréquenté

Objectif : comparer les informations publiées avec les besoins réels : entrée, circulation, ascenseur, toilettes, repos, contact, trajet et photos utiles.

5. Format envisagé

Le projet pourrait d’abord prendre la forme d’un site web simple, accessible et évolutif.

Ce site pourrait contenir :

  • une page de présentation du projet ;
  • des fiches pratiques par lieu, trajet ou service ;
  • des fiches téléchargeables en PDF accessible ;
  • une liste de ressources officielles ;
  • un formulaire de contribution citoyenne ;
  • quelques fiches pilotes documentées ;
  • une carte simple dans une seconde étape.

À moyen terme, une carte légère pourrait être envisagée, éventuellement en lien avec des outils existants comme OpenStreetMap, Wheelmap, Ginto ou d’autres plateformes d’accessibilité, si cela est techniquement et humainement réaliste.

Le format des fiches serait conçu pour pouvoir être réutilisé, croisé ou adapté plus tard avec d’autres plateformes ou données existantes : Ginto, Wheelmap, OpenStreetMap, données ouvertes de la Ville, ressources touristiques ou informations associatives.

6. Contribution citoyenne

Le projet pourrait permettre aux personnes concernées, proches, accompagnants, associations ou usagers de transmettre des informations pratiques.

Les contributions pourraient concerner :

  • un obstacle ;
  • un lieu accessible ;
  • une information incorrecte ;
  • une information manquante ;
  • un WC accessible ;
  • un parking PMR ;
  • un trajet difficile ;
  • un ascenseur en panne ;
  • des travaux ou obstacles temporaires.

Les contributions devraient être modérées avant publication afin de rester factuelles, respectueuses, utiles et juridiquement prudentes.

7. Durabilité et gouvernance

Pour que le projet reste utile dans le temps, il faut prévoir une forme de gouvernance simple.

Plusieurs modèles sont possibles :

  • projet indépendant et bénévole au départ ;
  • partenariat avec une association ;
  • petit comité citoyen de relecture ;
  • soutien ponctuel d’institutions ;
  • projet pilote financé ;
  • collaboration avec la Ville, Fribourg Tourisme ou des acteurs culturels.

La question centrale est : qui vérifie, met à jour, corrige et valide les informations sur le long terme ?

8. Accessibilité du guide lui-même

Le guide doit lui-même respecter les principes d’accessibilité.

Le futur site devrait donc être pensé avec :

  • bons contrastes
  • textes lisibles
  • navigation simple
  • compatibilité mobile
  • navigation clavier
  • images avec textes alternatifs
  • PDF accessibles
  • langage clair
  • chargement rapide

10. Mesure d’impact

Pour savoir si le projet fonctionne, plusieurs indicateurs simples pourraient être suivis :

  • nombre de fiches créées ;
  • nombre de lieux documentés ;
  • nombre de trajets documentés ;
  • nombre de WC accessibles recensés ;
  • nombre de parkings PMR documentés ;
  • nombre de contributions reçues ;
  • nombre de corrections apportées ;
  • retours d’utilisateurs ;
  • témoignages de personnes concernées ;
  • collaborations obtenues ;
  • améliorations concrètes réalisées par des lieux ou institutions.
